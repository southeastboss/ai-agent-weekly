# ai-agent-weekly 技术情报站设计文档

**日期：** 2026-03-25

## 1. 目标

把 `ai-agent-weekly` 从“泛 AI 新闻站”升级成“偏技术情报流 / 技术雷达站”。

新的核心目标：
- 不是泛泛汇总 AI 新闻
- 而是优先呈现**有技术价值、可执行、可用于判断方向**的内容
- 重点覆盖：开源项目、模型厂商动态、前沿技术

---

## 2. 新的信息架构

首页总数改为 **15 条**，分成 3 个分区：

### 2.1 开源项目（5 条）
聚焦真正有技术价值的项目，而不是单纯高 star 项目。

优先方向：
- Agent
- AI Coding
- MCP
- RAG
- Inference
- Eval
- Infra
- LLM 工具链

### 2.2 厂商动态（5 条）
覆盖全球头部模型厂商，但只保留技术密度高的动态。

重点厂商：
- OpenAI
- Anthropic
- Google
- xAI
- Meta
- 阿里
- 字节
- 百度
- MiniMax
- 智谱
- 月之暗面

### 2.3 前沿技术（5 条）
保留研究与工程结合紧密、对 Agent / 模型开发 / 部署 / 推理真正有帮助的内容。

---

## 3. 内容筛选原则

### 3.1 什么内容应该进入首页
满足以下任意一类即可优先入选：
- 能直接影响开发者或团队决策
- 能直接带来可用的新项目 / 新能力
- 能代表前沿技术趋势变化
- 对 Agent / Coding / 推理 / 评测 / 部署有实际帮助

### 3.2 什么内容应该降权或过滤
- 泛宏观 AI 新闻
- 纯市场 / 品牌 / 公关型动态
- 不含技术信息的采访或转述
- 技术价值弱的融资新闻
- 纯娱乐化 AI 内容

---

## 4. 各分区筛选逻辑

### 4.1 开源项目
保留“混合筛选，但只保留真正有技术价值的项目”。

衡量维度：
- 最近 24h / 7d 增长速度
- 技术相关 topic 命中情况
- 是否真实可用
- 是否有新意
- 是否不是简单套壳

### 4.2 厂商动态
只保留这些类型：
- 新模型 / 新版本发布
- API / SDK / Docs / Changelog 更新
- 价格变化
- 上下文长度 / 推理能力 / 工具调用 / Agent 能力变化
- 平台能力增强

### 4.3 前沿技术
优先保留：
- 有明确技术问题与解决方案的文章
- 工程可落地的研究或分析
- 对 Agent / 推理 / Eval / Infra 有直接帮助的内容

---

## 5. 排序逻辑

每个分区内部按综合分排序，而不是简单按时间排序。

建议综合分：

`综合分 = 技术价值 + 新鲜度 + 来源可信度 + 可落地性 + 相关度`

解释：
- **技术价值**：信息是否实质性
- **新鲜度**：是否是近期重要更新
- **来源可信度**：官方、优质技术媒体、优质开源仓库优先
- **可落地性**：用户是否能据此行动
- **相关度**：是否与 Agent / AI 开发工作流强相关

---

## 6. 页面展示方式

### 6.1 首页布局
首页改成 3 个明确版块：
- 开源项目
- 厂商动态
- 前沿技术

每个版块展示 5 条。

### 6.2 每条卡片字段
每条卡片建议展示：
- 原标题（保持原标题）
- 中文摘要（100 字内）
- 来源
- 时间
- 技术标签
- 价值标签（后续增强项）
- 图片（优先真实文章配图）

### 6.3 阅读顺序
建议页面上的阅读优先级为：
- 原标题
- 中文摘要
- 技术标签 / 价值标签
- 来源 / 时间

核心目标是让用户在 3 秒内判断“值不值得点进去”。

---

## 7. 数据源改造方向

### 7.1 开源项目源
新增 GitHub 项目源：
- GitHub Trending / 热门项目
- 指定 topic 仓库
- 高增量项目

### 7.2 厂商动态源
新增更接近一手信息的来源：
- 官方博客
- Release Notes
- Docs 更新页
- Changelog
- 官方发布页

### 7.3 前沿技术源
保留并优化现有来源：
- TechCrunch
- VentureBeat
- Hugging Face blog
- 其他高质量技术媒体 / 研究博客

---

## 8. 实施顺序

### Step 1：改页面骨架
- 总数改成 15 条
- 首页改成 3 个分区
- 保持原标题 + 中文摘要 + 来源 + 时间

### Step 2：改数据源
- 增加 GitHub 项目源
- 增加厂商官方动态源
- 降低泛媒体权重

### Step 3：改排序与质量规则
- 做技术价值打分
- 增加标签系统
- 做更稳的质量过滤

---

## 9. 推荐结论

推荐采用这个三阶段推进方案，原因：
- 不会一次性把现有站点推翻重做
- 每一步都可验证产出
- 先改结构，再改内容，再改质量，风险最低

最终目标不是做一个“更漂亮的 AI 新闻页”，而是做一个**值得每天看的 AI 技术情报站**。
